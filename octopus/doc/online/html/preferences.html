Octopus Preferences

You may customize Octopus using the Octopus preferences. These are accessable through the Window>Preferences menu. Select Octopus.
 You may set the following:
  • The default name for the model folder.
  • The default name for the expression folder.
  • The default visibility for attributes and association ends.
  • The default visibility for operations.
The first two may be overriden by the entries in the project properties of each individual project. The last two may be overriden by entering another visibility in the UML files.

Preferences for Code Generation

In a submenu to the Octopus preferences you can find the preferences for the Java Code Generation. For an explanation of the options see Code Generation Properties.
