New in version 1.1.1

1. There is an option to generate Java code from the model including code that implements the OCL expressions.
2. There is an XMI import option.
 3. And ... a number of bugs are fixed and the text of error messages has been improved.

New in version 1.1.0

Version 1.1.0 is the first release version of Octopus. Although some problems still remain (see Known Problems) but there is much improved in comparison to the previous beta versions. The differences between version 1.0.* and version 1.1.0 are the following:
 1. Version 1.1.0 is available under an open source license. An API to the parser/analyzer and to the abstract syntax tree (AST) that is build for each OCL expression by the parser/analyzer, is available in a seperate download.
 2. The Model View can be opened using the keyboard shortcut Ctrl+M.
 3. In the Model View you can select an element and open the file that contains the definition of this element by double clicking.
 4. Ocl Message expressions are correctly recognized.
 5. There is a separate view in which the abstract syntax tree (AST) of a selected OCL expression can be examined.
 6. There is a separate view in which the type hierarchy of a selected UML class or interface can be examined.
 7. You may customize Octopus through a number of preferences.
 8. You may change the names of the model and expression folder using the project properties.
 9. Multiplictities of [4..8] etc. are recognized.
 10. Self-refering associations, i.e. association from and to the same class, are supported, as well as self-refering association classes. (See The OCL Editor>Navigating Association Classes)
 11. The syntax for associations is a little bit stricter. Before there was no difference between the use of '-', '<-', '->', and '<->'. From this version the arrowheads are used to indicate navigability of the association end. (See The Textual Representation of UML.)
 12. To facilitate the mapping between the types used in your UML model and your OCL expressions, you can add a file called typeconversions.properties in the dedicated model folder. This file must contain a number of simple lines of the form modelType = oclType. (See The Textual Representation of UML.).
