Tips on the Use of Octopus

1. Let the folder structure under the expressions folder reflect the structure of your model. Use a different subfolder for each (sub)package.
 2. Use one OCL file for each class in the model.
 3. Use one UML file for each package in your base model.
 4. Keep additions to the UML elements in your base model in a file separate from the original UML file for that package.
 5. If the Model view does not update correctly, rebuild the project and try again.
 6. Move the Model view from the Tasks window to the Outline window. When Octopus is freshly installed, the Model View will open at the same position as the Tasks view. This can be tedious because you have to switch between Tasks and Model view often. Open the Outline view and drag the Model view on top of it. You can close the Outline view if you like. The Model view will now remain in the position of the Outline view.
 7. If an operation in a context definition cannot be found, check whether you have specified the correct return type. The return type should be present in the context declaration.
