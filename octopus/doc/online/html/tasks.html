How to ... in Octopus

What you want to do How you do it
Add elements to the UML Model from the Model View Double click on a package or class(ifier) in the Model View. The UML editor will open on a file called `NAME_additions', where NAME is the name of the selected item. If the file does not exist, it is created. Existing model elements will not be overwritten, provided they are not defined in this file.
Change default values for model and/or expression folders Open the Octopus preferences (Window>Preferences, select Octopus). Change the text fields and press the `Apply' or `OK' button.
Check OCL Expressions A `build' of an Octopus project will cause Octopus to check all expressions in files with extension `.ocl' in the dedicated expression folder in the project.
When the option `Perform build automatically on resource modification' is checked in the Workbench Preferences, a `refresh' or `save' of a file with extension `.ocl' in the dedicated expression folder in an Octopus project will cause Octopus to check all expressions in that file.
Create a Package in the UML Model Create a subfolder in the dedicated model folder. This will be considered a package in the model. Any files contained in this folder will be considered to specify elements within the package.
Create an Octopus Project Use the wizard: `New>Project>Octopus>NewOctopusProject'. Follow the directions.
Create Expressions from the Model View Double click on a class in the Model View. The Ocl Editor will open on the file called `NAME.ocl', where NAME is the name of the class. Enter your expressions on the class.
Create the UML Model from a modeling tool Use the Rose script or Poseidon XSL stylesheets to produce a set of files with extension .uml. Import these files in the dedicated model folder in the Octopus project.
Create or Change the UML Model Create a file with extension `.uml' in the dedicated model folder in the Octopus project. Write the model elements using the Octopus text notation for UML.
Edit Expressions from the Model View Double click on an OCL expression in the Model View. The Ocl Editor will open on the file that contains the expression.
Examine the internal representation of an OCL Expression Select the OCL expression in the Model View and open the context menu. From that select `Open AST Viewer'.
Examine the type hierarchy of a UML element Select the UML element in the Model View and open the context menu. From that select `Open Type Hierarchy'.
Filter Files with `.uml' Extension In Navigator View go to the `Filters' menu item (see Eclipse documentation). Check or uncheck the mark before `*.uml'.
Find an Error in an OCL Expression Double click on a task in the `Tasks' View. The OCL editor will open on the file that contains the error. (Standard Eclipse functionality)
Find the place where a model element has been defined Open the Model View and select a model element. Open the context menu on that element (right-click) and select 'Open definition'.
Generate Java code for the model and/or expressions Clicking on the Octopus icon in the toolbar will start the generation process on the selected project. If the button is not shown, you can use the Windows>Customize Perspective option to make Eclipse show it. Select Other>Octopus. Alternatively, click the Generate Java Code option in either the context menu on the project or the Octopus menu.
Rename the model and/or expression folder Open the project properties (select `Properties' from the context menu in the Navigator). Enter the new folder name and check the box 'Rename existing folder?' for the appropriate folder. Press the `Apply' or `OK' button.
Show the Business Modeling Syntax Open the file with the Ocl Multipage Editor. In the second tab, called `Business Modeling View' the expressions are shown in the Business Modeling Syntax. In this view the expressions are not editable.
Another possibility is to open the file with the BM Editor. The expressions are shown in the Business Modeling Syntax. In this editor the expressions are not editable.
Show the Model Clicking on the Octopus icon in the toolbar will open the Model View on the selected project. If the button is not shown, you can use the Windows>Customize Perspective option to make Eclipse show it. Select Other>Octopus. Alternatively, click the menu item `Octopus>Show Model' in the context menu of a file with extension `.ocl' or `.uml', or the context menu of an Octopus project. A third alternative is to select a file or folder in an Octopus project and type Ctrl+M on your keyboard
Use Code Completion Either use `Ctrl-Space, or the menu item "Edit>Content Assist".
