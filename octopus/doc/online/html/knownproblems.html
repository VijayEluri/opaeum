Known Problems and Omissions

The current version of Octopus is the first release version. The following issues are known to us. If you encounter any other problems, please inform us by sending an email to octopus@klasse.nl.

User Interface Issues

1. The keybindings in the Ocl Multipage Editor do not function. Use the single page Ocl Editor.
 2. The icons in the Model View and Type Hierarchy View are sometimes shown twice as large.
 3. The buttons in the tool bar do not appear when Octopus is loaded the first time. Use Window>Customize Perspective>Other to make them appear.

Modeling Issues

1. Octopus does not recognize inner classes, i.e. classes that are owned by a class.
 2. Octopus does not take into account the visibility of the model elements. Each element is considered to be visible to all other elements.
 3. Octopus does not allow multiple package statements in one file.
 4. Octopus does not recognize components.
 5. Qualifiers are not recognized.

Code Generation

1. When the option Generate interfaces and 'internal' package is selected and the code generation process is started more than once, the code generation does not clean the generated interfaces correctly. For now you have to remove them by hand.
 2. You cannot use both the options Generate interfaces and 'internal' package and Generate user interface prototype. The generated user interface does not recognise the generated interfaces correctly.
 3. When you press the button to generate code while the currect selection is not an element in the Package Explorer or Navigator, or the element is not a project, a thread access exception occurs.
 4. When reading objects that have been stored using the generated storage facility, the structure of the XML file is not checked against the XML schema.
 5. The preferences for code generation are not taken into account correctly. You have to open the project properties and click 'Apply' or 'OK'.

XMI import

1. Poseidon generates XMI files in which there are operations that have a return type, but this type does not have a name. These operations are not imported correctly. We consider this to be a flaw in the Poseidon XMI generation.
