The OCL Editor

The OCL Editor is a simple editor for OCL files. It offers syntax highlighting and a rudimentary form of code completion. It can be opened on any file with extension '.ocl'.

The structure of an OCL file

In version 2.0 of OCL the OCL file structure was introduced. As in earlier versions the context definition of an OCL expression specifies which model elements are visible in the expression. To avoid having to write the path of the enclosing package over and over again, an OCL file may contain a package-endpackage statement. The path of the package mentioned in the package statement is considered to be prefixed to the path in the context definition. Thus the following two OCL files are semantically equivalent.

-- file 1
 package Package::SubPackage
context X
inv:... some invariant ...
 context X::operationName(..)
 pre: ... some precondition ...
endpackage

-- file 2
 context Package::SubPackageX
inv:... some invariant ...
 context Package::SubPackageX::operationName(..)
 pre: ... some precondition ...

According to the standard an OCL file (or stream) may contain any number package statements. This version of Octopus allows only one package statement per file.

Navigating Association Classes

To navigate from either side of an association to the association class you may use the name of the association class. For instance, using the model trom the figure, the following expression is valid.
context Company
 inv: Job->notEmpty()

However, when the class at both ends of the association is the same, this results in an ambiguity. For instance, using the same model, the following expression is ambiguous:

context Person
 inv: Marriage->notEmpty()

When navigating from the wife role, this statement would mean something like: "every person must have a husband". When navigating from the husband role, this statement would mean something like: "every person must have a wife".

To make explicit from which end you navigate to the association class, you must adorn the association class name with the role name at the opposite end. For instance:

context Person
 inv: Marriage[wife]->notEmpty()
