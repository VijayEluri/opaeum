The Model View

The Model View is a window showing the complete model in a tree hierarchy. It can be opened by clicking on the Octopus icon in the Navigator, or be selecting the context menu option 'Octopus>Show Model'. This menu option is available on UML and OCL files and on Octopus projects.

In the Model View the complete model is shown, that is, the elements defined in the UML files and the elements defined in the OCL files are shown in one hierarchy. The base elements like classes, attributes and operations, etc. come from the UML files. Note that associations are shown in two manners:

1. as association under the package that owns it
 2. as 'navigation' under the class that is part of an association
 From the OCL files the following elements are included in the model hierarchy:
  • attributes defined in an OCL expression, shown both in the attribute list and as definition expression under the class that owns the attribute,
  • operations defined in an OCL expression, shown both in the operation list and as definition expression under the class that owns the operation,
  • invariants,
  • pre- and postconditions of operations,
  • the body expression of an operation,
  • initial values and derivation rules for attributes,
  • initial values and derivation rules for navigations.

Opening the Definition of an Element

From the Model View you can open the file that holds the definition of the selected model element. You can open the definition by
1. double clicking, or
 2. selecting the context menu 'Open Definition'.
If the element is an OCL expression the OCL editor will be started, if the element is defined in a UML file the UML editor is started.

Opening the OCL Editor from the Model View

The OCL editor will be opened by clicking the OCL file icon, or by selecting the context menu 'Open/Create OCL File'.
If a base element (an element defined in a UML file) is selected at the moment that the OCL editor is opened, it will open on a file called 'NAME.ocl', where NAME is the name of the selected element. Note that this will only work for Classifiers: classes, datatypes, enumerations, and interfaces. If the file did not yet exist, it will be created and it will be placed in a directory with the name of its enclosing package. If the directory does not exist, it will be created also.

Opening the UML Editor from the Model View

From the Model View you can also open the UML editor. You can open the UML editor by selecting the context menu 'Add UML Elements', or clicking the UML file icon. Because Octopus assumes that the original base model should be kept unchanged, all changes (in fact, additions) to the base model are kept in separate UML files. The editor will therefore be opened on a special file, called 'NAME_additions.uml', where NAME is the name of the element that was selected in the Model View. Note that for this purpose you can only select packages or classes. If the file did not yet exist, it will be created.
