An Octopus Project

The functionality provided in Octopus is available only on projects with an Octopus nature. To create a project with this nature, use the wizard: `New>Project>Octopus>NewOctopusProject' and follow the directions.

Adding the Octopus Nature to a Java Project

The Octopus nature may also be added to a Java project (to be precise, a Java project is a project with the Java nature). To do so either use the `Octopus' context menu on the Java project and select `Add Octopus Nature', or use the button with the red `O'. Another way to start code generation is to select the `Add Octopus Nature' entry in the Octopus menu.

Model and Expression Folders

The NewOctopusProject wizard will ask you to give the names of two special folders within the project. The names can be changed later on using the project properties.

  • The 'folder for UML model', in short the 'model folder', is the folder where the files that specify the UML elements that are used when writing OCL expressions, are stored.
  • The 'folder for OCL expressions', in short the 'expression folder', is the folder where the files that contain OCL expressions are stored.
When the Octopus nature is added to a Java project, the names for both directories are taken from the preferences. As with ordinary Octopus projects, the names can be changed later on using the project properties.
